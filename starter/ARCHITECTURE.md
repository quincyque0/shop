# Архитектура проекта "Магазин товаров"

## Описание
Проект представляет собой систему для интернет-магазина, состоящую из нескольких независимых микросервисов, взаимодействующих между собой через синхронные и асинхронные протоколы.

### Микросервисы
1. **API Gateway**: Единая точка входа для всех внешних клиентов. Маршрутизирует HTTP запросы к нужным сервисам.
2. **Catalog Service**: Управляет каталогом товаров (создание, получение списка, проверка наличия).
3. **Order Service**: Управляет заказами. Проверяет наличие товара через Catalog Service и инициирует асинхронную отправку уведомлений.
4. **Notification Service**: Служба рассылки уведомлений пользователям (имитация).

## Взаимодействие
- **Внешние клиенты** общаются с API Gateway по HTTP (REST).
- **API Gateway** общается с сервисами Catalog и Order по HTTP (REST).
- **Order Service** общается с **Catalog Service** по **gRPC** для быстрой проверки наличия товара при создании заказа.
- **Order Service** общается с **Notification Service** асинхронно через брокер сообщений **RabbitMQ** (публикует событие `OrderCreated`).

## Инфраструктура
- **Базы данных**: PostgreSQL. `catalog-svc` и `order-svc` используют свои логические базы данных (`catalog_db`, `orders_db`) в одном инстансе PostgreSQL.
- **Очереди сообщений**: RabbitMQ.
- **Контейнеризация**: Все компоненты (сервисы, БД, RabbitMQ) упакованы в Docker и запускаются через `docker-compose up`.

## Технологический стек
- Язык: Python 3
- Фреймворки: FastAPI (REST), grpcio (gRPC)
- ORM/БД драйверы: SQLAlchemy, asyncpg
- Брокер: RabbitMQ (pika / aio_pika)
